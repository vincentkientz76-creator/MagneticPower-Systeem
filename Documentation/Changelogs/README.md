# Changelogs — Wijzigingen & Versiebeheer

Deze map bevat alle wijzigingen binnen het MagneticPower Systeem.

## Waarom changelogs?

- Het systeem moet auditbaar blijven.
- Elke wijziging moet herleidbaar zijn.
- De DGA moet altijd kunnen zien wat er is veranderd.
- Agents moeten weten welke regels zijn aangepast.

## Structuur

Elke wijziging krijgt:

- Datum  
- Auteur (DGA of agent)  
- Betrokken regels  
- Beschrijving van de wijziging  
- Impactanalyse  
- Status (GO / NO‑GO)  

## Richtlijnen

- Geen wijzigingen zonder DGA‑besluit.  
- Geen wijzigingen zonder changelog‑entry.  
- Geen wijzigingen buiten deze map.
